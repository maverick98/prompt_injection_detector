# Prompt Injection Detector Dataset Card

## Dataset Summary

This repository includes a reproducible bootstrap dataset for prompt-injection
detection research. The generated local dataset contains 1,500 examples:

- 750 prompt-injection attempts
- 750 clean prompts
- 70/15/15 train/validation/test split
- five injection categories plus clean traffic

The dataset is generated by deterministic templates in
`src/prompt_injection_detector/data/synthetic.py`. It is designed to make the
pipeline runnable without external credentials, not to replace a manually reviewed
research corpus.

## Labels

| Column | Meaning |
|---|---|
| `text` | Prompt, document excerpt, or user input |
| `label` | `1` for injection, `0` for clean |
| `category` | Injection category or `clean` |
| `source` | Origin of the row |
| `split` | `train`, `val`, or `test` |

## Injection Categories

| Category | Description |
|---|---|
| `role_override` | Attempts to make the model ignore higher-priority instructions or adopt a new role |
| `instruction_smuggling` | Hides malicious instructions inside normal-looking content |
| `data_extraction` | Attempts to reveal system prompts, memory, or confidential context |
| `jailbreak` | Uses role-play, hypotheticals, or framing to bypass safety boundaries |
| `indirect_injection` | Embeds instructions inside documents, web pages, emails, or retrieved content |

## Recommended Expansion

For a CV-grade or publishable version, augment the starter data with:

- public prompt-injection datasets such as HuggingFace `deepset/prompt-injections`
- OWASP LLM Top 10 examples
- responsibly gathered jailbreak examples
- manually crafted examples for weak categories
- LLM-generated examples followed by human review

## Known Limitations

- Synthetic templates are easier than real-world attacks.
- Current examples are primarily English.
- No proprietary prompts, real secrets, or production system prompts are included.
- Metrics on the starter split should be treated as pipeline validation, not as
  proof of production robustness.

## Intended Use

This dataset is intended for defensive research, benchmarking, education, and
portfolio demonstration of prompt-injection detection workflows.

## Out-of-Scope Use

Do not use this dataset to attack deployed systems, collect confidential prompts,
or bypass real safety controls.
