Executive Summary

The project is a defensive research platform for detecting prompt-injection attacks against LLM applications. It is intentionally not just a binary classifier. It combines model-based detection, red-team attack generation, adversarial retraining, hard-case evaluation, minimax attacker/defender analysis, and an interpretable mathematical risk layer that explains why a prompt may be dangerous.

Security Goal

Reduce false negatives on prompt-injection attempts while preserving enough precision for usable guardrails.

Research Goal

Show that detectors and attackers should co-evolve rather than treating prompt injection as a static dataset problem.

Explainability Goal

Give interpretable scores: leakage pressure, risk energy, uncertainty, graph paths, and recommended control action.

Delivery Goal

Provide CLI workflows, reports, a Colab notebook, Streamlit Cloud app, and visual documentation.

Core thesis

Prompt injection is adversarial behavior, so a strong solution should evaluate both the current detector and the adaptive attacker. The best product posture is not just classification; it is classification plus calibrated risk, uncertainty, and control.

System Architecture

The architecture is built as a pipeline with feedback loops. Inputs become features and risk signals. The detector produces a probability and category. The research layer produces mathematical explanations. Red-team components mutate attacks to expose blind spots. Evaluation modules generate hard-suite metrics and game-theoretic equilibrium reports.

Classical Detector

The default detector is deliberately lightweight, local, and reproducible. It uses TF-IDF features over both words and character n-grams. Character n-grams help detect obfuscation, spacing tricks, encodings, and misspellings often used in prompt-injection attacks.

Feature Union

Word n-grams capture instruction semantics. Character n-grams capture obfuscation and partial attack strings.

features = TFIDF_word(1..3) + TFIDF_char(3..5)

Model Selection

Logistic regression, calibrated SVM, and random forest are compared. The chosen model prioritizes recall then F1.

argmax_model (recall_injection, F1_injection)

Threshold Calibration

The threshold is tuned for security posture: false negatives are more dangerous than false positives.

detect if score >= threshold
Security interpretation

In prompt-injection defense, recall is not an ordinary metric. A missed attack can expose secrets, tools, or hidden instructions. Precision still matters, but recall-first calibration is the safer initial operating mode for high-risk LLM applications.

Red Team And Adversarial Loop

The red-team module turns seed attacks into evasion variants, then scores each variant with the detector. The adversarial loop uses missed attacks to retrain and re-evaluate the detector, creating an attack-defend-evolve cycle.

Paraphrase

Rewords the attack while preserving adversarial intent.

Obfuscation

Uses spacing, substitutions, and Unicode-like tricks to avoid simple keyword matching.

Role Wrapping

Frames the attack as role-play, debugging, policy update, or administrative override.

Encoding

Hides instructions in base64-like or encoded-looking forms.

Nested Injection

Places adversarial instructions inside quoted text, documents, or simulated retrieved content.

Retraining

Adds successful evasions back into training data and repeats evaluation.

for each round: train -> find misses -> generate evasions -> add successful variants -> retrain

Game-Theoretic Attacker/Defender Analysis

Prompt injection is strategic: attackers choose evasion strategies, defenders choose detection thresholds and guardrail policies. We model this as a finite zero-sum game where the defender minimizes worst-case loss.

Attacker Actions

Paraphrasing, obfuscation, role-play, encoding, nested injection, and other red-team variants.

Defender Actions

Threshold policies that trade bypass prevention against false-positive burden.

defender_loss = bypass_rate + false_positive_weight * false_positive_rate min_defender max_attacker loss(strategy, threshold)
Why this is useful

Game theory prevents a detector from optimizing only against yesterday's attack. It asks: if the attacker adapts, which defense policy still performs well?

Mathematics, Physics, And Information Theory

The new research layer is designed as interpretable analysis beside the classifier. It does not pretend that physics magically solves prompt injection. Instead, it uses rigorous ideas as useful analogies and scoring tools for risk, uncertainty, and control.

1. Information Theory

We estimate whether a prompt is attempting to extract hidden state. Signals include secret-related words, hidden-context references, entropy, and compression pressure.

H(X) = - sum p(x) log2 p(x) leakage_score = f(secret_terms, hidden_state_terms, entropy)

2. Statistical Physics / Energy Risk

Each prompt is mapped to a risk-energy landscape. Higher energy means stronger pressure toward unsafe behavior: override, exfiltration, tool abuse, obfuscation, or authority hijack.

E(prompt) = w1*override + w2*exfiltration + ... + wk*leakage free_energy = -log(1 - E)

3. Phase Transition Bands

Risk changes are treated like operating regions. Low-risk prompts pass normally, critical prompts are reviewed, and high-risk prompts are blocked or quarantined.

E < 0.35: allow | 0.35 <= E < 0.62: review | E >= 0.62: block

4. Conformal-Style Uncertainty

When the classifier score or energy score sits near a boundary, the system marks the prediction set as ambiguous instead of pretending to be overconfident.

prediction_set = {clean, injection} near decision boundary

5. Sequential Detection

Multi-turn attacks may distribute risk over several harmless-looking messages. A CUSUM-like statistic accumulates risk energy across turns.

S_t = max(0, alpha*S_(t-1) + E_t - baseline)

6. Graph Theory

RAG and agent attacks can be viewed as dangerous paths: untrusted content influences instructions, instructions cross a policy boundary, hidden context flows to a tool or sink.

untrusted_content -> instruction_override -> hidden_context -> external_sink

7. Optimal Transport Intuition

The risk vector is compared to prototype clean and attack distributions. If it is closer to the attack prototype, its attack affinity rises.

attack_affinity = d(clean) / (d(clean) + d(attack))

8. Control Theory

The detector acts like a controller that keeps an LLM application inside a safe operating region. Actions are allow, review, quarantine, and block.

control_signal = max(E, graph_risk, uncertainty_pressure)
